## 一、当前阶段评估

### 1.1 项目成熟度总览

| 维度 | 当前原型状态 | 量产商业化要求 | 达成度 | 核心差距 |
|------|-------------|---------------|--------|---------|
| **软件健壮性** | 双核去耦、零内存分配、自动恢复 | 常开无崩溃、看门狗覆盖、数据完整 | **95%** | 微调 AP 配网安全性 |
| **PC 端体验** | Python 脚本运行，终端输出 | 独立安装包（EXE），图形化一键配置 | **60%** | PyInstaller 打包 + 防火墙自动放行 |

### 1.2 软件与固件已达到的工业级指标

在纯软件与协议层面，以下设计已达到工业级标准，保障设备可以 **7x24 小时常开不假死**：

1. **高并发与零碎片内存管理**：固件端彻底摒弃运行时动态 `malloc`，采用 PSRAM 静态块缓冲区复用（`g_pxlPool` 和 `g_jsonParseBuf`），规避嵌入式设备因内存碎片化导致的 OOM 崩溃。
2. **双核 SMP 架构无锁异步拓扑**：Core 0 负责网络通信及 JSON 协议流式解析，Core 1 负责 LCD 硬件刷新与交互，两核通过 `std::atomic` 指针和双缓冲进行状态交换，完全避免互斥锁死锁及画面卡顿。
3. **多级主动省电与射频热控制**：系统在检测到静默后，自动在 "全亮 60fps → 变暗 5fps → 休眠 2fps + Modem-Sleep → Light Sleep" 之间平滑切换，在保障 WiFi TCP 常开的同时大幅降低待机功耗与射频发热。
4. **通信层的高容错抗抖动机制**：PC 端采用线程安全的异步发送队列，辅以带随机抖动的指数退避重连算法，防止网络闪断导致主程序假死；固件端支持物理层 Keep-Alive 检测与自动复位。

### 1.3 商业落地的客观瓶颈

#### 硬件级瓶颈

```
当前开发板方案 (约 ¥120+)              量产定制 PCB 方案 (目标 ¥35-45)
┌──────────────────────────┐           ┌──────────────────────────┐
│ Waveshare S3 开发板 (¥70)│           │ ESP32-S3 核心芯片 (¥12)  │
│ 各种外接排针传感器 (¥40) │   ───►    │ 1.54寸 LCD 裸屏 (¥8)     │
│ 杜邦线/面包板连接        │           │ 传感器电路集成 (¥10)     │
│ 3D 打印外壳 (易积热)     │           │ 模具注塑外壳 (¥5)        │
└──────────────────────────┘           └──────────────────────────┘
```

- **BOM 成本过高**：使用微雪等第三方集成开发板，单台硬件成本 120 元以上，无法支撑消费电子的商业利润空间。
- **物理连接隐患**：杜邦线、外部模块插接等连接方式抗震性极差，在物流运输或日常桌面震动中极易发生 I2C 总线断开或引脚虚接。
- **认证与合规**：作为无线射频消费电子设备，上市必须通过 SRRC、FCC、CE 等强制认证，现有散件拼装方案无法通过 EMC/EMI 测试。

#### 软件级瓶颈

- **运行依赖高**：PC 端基于 Python 源码运行，要求用户安装 Python 环境、pip 依赖、配置 `config.json`，对非技术用户构成不可逾越的门槛。
- **网络安全隔离**：PC 端 TCP 端口绑定在 `0.0.0.0`，局域网内任何人都可向用户电脑发送伪造数据，存在被嗅探或攻击的安全风险。

## 二、DFM 可制造性设计

### 2.1 PCB 单板集成方案

#### 设计目标

将当前散件拼装方案整合为单块 PCB，消除所有杜邦线连接。

#### 电路集成清单

| 模块 | 当前方案 | 量产集成方案 | 节省 |
|------|---------|-------------|------|
| 主控 | ESP32-S3 开发板 | ESP32-S3FN8 裸片 QFN56 | - |
| 屏幕 | 开发板集成 | FPC 连接器 (8pin, 0.5mm间距) | - |
| 光照传感器 | BH1750 模块 | BH1750FVI 贴片 (SOP-8) | 模块封装 |
| 触觉马达 | DRV2605L 模块 | DRV2605L 贴片 (SOP-10) | 模块封装 |
| 蜂鸣器 | 无源蜂鸣器模块 | 贴片蜂鸣器 (1206) | 模块封装 |
| 触摸 | TTP223 模块 | ESP32 电容触摸引脚 (GPIO4) | 整个模块 |
| 编码器 | EC11 模块 | EC11 直焊 PCB | 杜邦线 |

#### PCB 层叠推荐

**量产推荐 4 层板**：
```
Layer 1 (Top)    : 信号 + 器件
Layer 2 (Inner1) : GND 完整平面
Layer 3 (Inner2) : 电源平面
Layer 4 (Bottom) : 信号 + 器件
```

**打样验证阶段可先用 2 层板**：
```
Layer 1 (Top)    : 信号 + 器件 + 局部铺铜 GND
Layer 2 (Bottom) : GND 铺铜 + 少量信号
```

#### 关键布局规则

1. **晶振区域**：ESP32 晶振下方保持 GND 铜皮完整，禁止走线
2. **天线区域**：PCB 天线区域禁止铺铜、禁止放置器件，预留净空区
3. **高速信号**：SPI 走线长度匹配，阻抗控制 50 欧姆
4. **电源走线**：主电源线宽 ≥ 0.5mm，建议 1mm
5. **LCD 连接器**：FPC 连接器靠近板边，便于安装
6. **螺丝孔**：四角 M2 螺丝孔，直径 2.2mm

#### 参考布局

```
┌──────────────────────────────────────┐
│  USB-C    [LDO]    [ESP32-S3]        │
│           [电容]     │               │
│                      │    [天线区]   │
│  [FLASH]  [晶振]    │    (净空)     │
│                      │               │
│                      │               │
│         [LCD FPC连接器]              │
│  [EC11]  [蜂鸣器]                    │
│  [LRA马达]                           │
└──────────────────────────────────────┘
```

### 2.2 外壳设计

#### 阶段一：3D 打印验证（EVT/DVT 阶段）

- 材料：PLA 或 PETG
- 工艺：FDM 打印
- 成本：¥5~15/个（小批量）
- 周期：1~3 天
- 用途：验证装配、散热、按键手感

#### 阶段二：注塑量产（PVT/MP 阶段）

- 材料：ABS 或 PC（耐热、抗冲击）
- 工艺：注塑成型
- 模具费用：¥5,000~20,000（单穴模具）
- 单件成本：¥3~8/个（批量 1000+）
- 周期：模具制作 15~30 天

#### 外壳设计要点

| 设计要素 | 要求 | 原因 |
|---------|------|------|
| **导光柱** | 正面为 BH1750 预留半透明亚克力窗 | 确保环境光无折射损失抵达传感器 |
| **散热孔** | 底部开直径 2mm、间距 5mm 的散热孔 | 防止 ESP32-S3 在 240MHz WiFi 传输时积热 |
| **LCD 固定** | 双面胶 + 卡扣 | 确保屏幕不晃动 |
| **USB 开口** | 预留 Type-C 接口位 | 供电/烧录/调试 |
| **编码器开口** | 侧面预留旋钮伸出位 | 用户交互 |
| **触摸区域** | 顶部预留触摸感应区 | 需保持一定厚度（≤3mm） |

## 四、合规认证清单

### 4.1 无线认证

作为集成 WiFi + BLE 的消费电子设备，无线认证是上市的**法规红线**。

| 认证 | 适用地区 | 测试内容 | 预计费用 | 周期 |
|------|---------|---------|---------|------|
| **SRRC** | 中国（强制） | 无线电型号核准，射频参数测试 | ¥15,000~25,000 | 4~8 周 |
| **FCC (ID)** | 美国 | Part 15 射频辐射测试 | ¥20,000~35,000 | 6~10 周 |
| **CE (RED)** | 欧盟 | 无线设备指令，射频+安全 | ¥25,000~40,000 | 6~12 周 |
| **MIC** | 日本 | 技术基准适合证明 | ¥15,000~25,000 | 4~8 周 |

> **重要提示**：使用 ESP32-S3 模组（已通过 FCC/CE/IC/UKCA 认证）可大幅降低整机认证难度和费用。模组认证号可用于整机认证的子模块引用。

| 辐射发射 | CISPR 32 / EN 55032 | 空间辐射干扰 |
| 静电放电 (ESD) | IEC 61000-4-2 | 接触 ±4kV，空气 ±8kV |
| 浪涌抗扰 | IEC 61000-4-5 | 电源线浪涌 |

## 六、产测方案

### 6.1 硬件自检流程

固件内置工厂测试模式，通过特定 GPIO 拉低触发（或 NVS 未写入 `TEST_PASS` 标记时自动进入）。

```
上电
 │
 ▼
┌─────────────────────────────────────────────────┐
│              工厂测试模式 (FCT)                   │
│                                                  │
│  1. LCD 测试: 全红 → 全绿 → 全蓝 → 白 → 黑      │
│     → 检查: 无坏点、色彩准确                      │
│                                                  │
│  2. 背光测试: 0% → 50% → 100% → 0%              │
│     → 检查: PWM 调光平滑                         │
│                                                  │
│  3. I2C 扫描: 检测 BH1750 (0x23) + DRV2605L (0x5A)│
│     → 检查: 地址正确，通信正常                     │
│                                                  │
│  4. 触觉测试: 触发 click/buzz 振动                │
│     → 检查: 马达振动正常                          │
│                                                  │
│  5. 蜂鸣器测试: 播放测试音阶                      │
│     → 检查: 声音正常                              │
│                                                  │
│  6. 触摸测试: 等待触摸事件                        │
│     → 检查: 触摸响应正常                          │
│                                                  │
│  7. 编码器测试: 等待旋转+按压事件                 │
│     → 检查: 旋转方向正确，按压响应正常             │
│                                                  │
│  8. WiFi 测试: 扫描测试 AP (车间专用)             │
│     → 检查: 信号强度 > -70dBm                     │
│                                                  │
│  9. 光照测试: 读取 BH1750 数值                    │
│     → 检查: 车间光照下数值在合理范围               │
│                                                  │
│  全部通过 → NVS 写入 TEST_PASS → 正常启动         │
│  任一失败 → NVS 写入 TEST_FAIL:{错误码} → 报错    │
└─────────────────────────────────────────────────┘
```

## 七、质量控制

### 7.1 老化测试

#### 测试方案

| 测试类型 | 条件 | 持续时间 | 抽样比例 |
|---------|------|---------|---------|
| 高温老化 | 60°C 恒温箱 | 48 小时 | 首批 100%，后续 10% |
| 低温启动 | -10°C 冷藏 | 24 小时 | 首批 50%，后续 5% |
| 温度循环 | -10°C ~ 60°C，2小时/循环 | 10 个循环 | 首批 50%，后续 5% |
| 电源波动 | 4.5V ~ 5.5V 供电 | 24 小时 | 首批 50%，后续 5% |
| 长期运行 | 常温常湿，持续运行 | 7 天 | 首批 100%，后续 5% |

#### 老化测试检查项

- [ ] 无死机/重启
- [ ] WiFi 连接保持稳定
- [ ] LCD 显示无异常（坏点、色偏、亮度衰减）
- [ ] 内存无泄漏（堆内存稳定）
- [ ] 温度在安全范围内（<65°C）
- [ ] 电流在规格范围内

### 7.2 抽检标准

#### AQL 抽检方案（参照 GB/T 2828.1）

| 检验项目 | 检验水平 | AQL 值 | 说明 |
|---------|---------|--------|------|
| 外观检查 | II | 1.0 | 划痕、色差、装配间隙 |
| 功能测试 | II | 0.65 | 全功能验证 |
| 电气参数 | S-2 | 0.4 | 电流、电压、功耗 |
| 射频参数 | S-1 | 0.25 | 发射功率、接收灵敏度 |

#### 批次记录

每批次需记录：
- 生产日期、批次号
- BOM 版本、固件版本
- SMT 炉温曲线
- 测试通过率
- 不良品分析

### 7.3 OTA 售后升级

#### OTA 架构

```
┌──────────┐     HTTPS      ┌──────────┐
│  ESP32   │ ◀──────────────│ OTA      │
│  设备     │  固件下载       │ 服务器   │
└──────────┘               └──────────┘
     │                           │
     │  检查更新                  │  固件管理
     ▼                           ▼
┌──────────┐               ┌──────────┐
│ 当前版本  │               │ 版本列表  │
│ 检查间隔  │               │ 灰度发布  │
│ 回滚机制  │               │ 强制升级  │
└──────────┘               └──────────┘
```

#### OTA 安全机制

| 机制 | 说明 |
|------|------|
| **固件签名** | 每个固件使用私钥签名，设备端用公钥验证 |
| **版本校验** | 防止降级攻击，只允许升级到更高版本 |
| **A/B 分区** | 升级失败自动回滚到上一版本 |
| **差分升级** | 只传输差异部分，节省流量和时间 |
| **断点续传** | 网络中断后可从断点继续下载 |

#### OTA 流程

```
1. 设备定时检查 OTA 服务器（间隔 24 小时）
2. 服务器返回最新版本号和固件摘要
3. 设备比较版本号，若有新版本则下载固件
4. 验证固件签名和 CRC
5. 写入 B 分区
6. 重启切换到 B 分区
7. 新固件自检通过 → 确认升级
8. 新固件自检失败 → 自动回滚到 A 分区
```

### 7.4 售后支持

#### 问题分级

| 级别 | 描述 | 响应时间 | 解决方案 |
|------|------|---------|---------|
| P0 | 设备无法启动/变砖 | 4 小时 | 远程指导重刷固件 |
| P1 | 核心功能不可用 | 24 小时 | OTA 紧急修复 |
| P2 | 次要功能异常 | 72 小时 | 下一版本修复 |
| P3 | 体验优化 | 下个迭代 | 需求排期 |

#### 远程诊断

通过 MQTT 遥测数据，可远程获取：
- 设备固件版本
- 运行时长和重启次数
- WiFi 信号强度和断连次数
- 堆内存使用趋势
- 温度和电压
- 最近的错误日志
