# 像素动画制作教程

本教程面向桌面宠物 ESP32 项目，指导如何制作高质量的 32x32 像素动画，并转换为 `.pxl` 格式推送到设备显示。

---

## 一、32x32 像素画设计原则

### 1.1 有限画布的表达策略

32x32 共 1024 个像素点，空间极为有限。设计时应遵循以下原则：

- **大胆简化**：放弃细节，保留核心轮廓和特征。例如人脸只需 2-3 个像素表示眼睛、1 个像素表示嘴巴。
- **高对比度**：在小尺寸屏幕上，低对比度的颜色差异几乎不可见。优先使用明暗对比强烈的配色。
- **粗线条**：线条宽度至少 2 像素，1 像素线条在屏幕上容易断裂或闪烁。
- **留白**：不要填满整个画布，适当的留白让主体更突出。建议主体占 20x20 ~ 28x28 的区域。

### 1.2 色彩限制

ESP32 使用 RGB565（16bit）色彩，R/B 各 5 位（32 级）、G 6 位（64 级）。设计时注意：

- 避免大面积渐变，容易出现色带
- 同一画面建议控制在 8-16 种颜色以内
- 纯黑 `#000000`、纯白 `#FFFFFF` 在 RGB565 下完全无损
- 常用的安全色系：红 `#FF0000`、绿 `#00FF00`、蓝 `#0000FF`、黄 `#FFFF00`、橙 `#FF8800`

### 1.3 动画帧数建议

| 类型 | 帧数 | 帧间隔 | 说明 |
|------|------|--------|------|
| 静态图标 | 1 | 200ms | 表情、符号、Logo |
| 简单动画 | 2-4 | 200-300ms | 闪烁、呼吸、点头 |
| 行走动画 | 4-8 | 100-150ms | 角色走路循环 |
| 复杂动画 | 8-16 | 80-120ms | 跳舞、表情变化 |

> **上限**：PXL 格式最大支持 64 帧动画，单文件约 128KB。实际使用中 16 帧以内即可满足大多数需求。

## 二、推荐工具

### 2.1 Aseprite（首选）

- **类型**：专业像素画编辑器（付费，可自行编译）
- **平台**：Windows / macOS / Linux
- **优势**：动画时间轴、洋葱皮、调色板管理、导出 Spritesheet
- **导出**：File > Export Spritesheet > 水平排列，格式 PNG

### 2.2 Piskel（免费在线）

- **类型**：浏览器端像素画编辑器
- **网址**：https://www.piskelapp.com
- **优势**：免费、无需安装、支持动画预览、可导出 GIF 和 Spritesheet
- **适用**：快速原型、简单动画

### 2.3 LibreSprite（免费开源）

- **类型**：Aseprite 的开源分支（停留在旧版本）
- **平台**：Windows / macOS / Linux
- **优势**：完全免费、功能与早期 Aseprite 类似
- **适用**：预算有限但需要桌面客户端的用户

### 2.4 工具选择建议

| 需求 | 推荐工具 |
|------|----------|
| 专业像素动画制作 | Aseprite |
| 快速在线编辑 | Piskel |
| 免费桌面客户端 | LibreSprite |
| 仅需简单静态图 | 任意绘图软件（设置 32x32 画布） |

## 三、静态图片制作流程

### 3.1 创建画布

在绘图工具中新建 32x32 像素的画布。以 Aseprite 为例：

1. File > New > 宽度 32、高度 32
2. 关闭背景透明（Background: Transparent 或 White）
3. 缩放视图至 800%-1600% 以便编辑

### 3.2 绘制

- 使用铅笔工具（Pencil），笔刷大小 1-2 像素
- 利用对称工具加速对称图形的绘制
- 使用填充工具（Fill/Bucket）快速上色

### 3.3 导出

将完成的像素画导出为 PNG 格式：

- Aseprite: File > Export As > PNG
- Piskel: 下载 PNG（Download PNG）
- 通用绘图软件: 另存为 PNG，确保无缩放

### 3.4 转换为 PXL

```bash
python pixel_tool.py convert heart.png -o heart.pxl
```

使用 `info` 验证：

```bash
python pixel_tool.py info heart.pxl
```

## 四、动画 GIF 制作流程

### 4.1 逐帧绘制

1. 在 Aseprite 或 Piskel 中创建 32x32 画布
2. 逐帧绘制动画的每个状态
3. 利用洋葱皮（Onion Skin）功能查看前后帧的叠影，确保动画流畅
4. 设置帧延迟时间（建议 100-200ms）

### 4.2 导出 GIF

- Aseprite: File > Export As > GIF
- Piskel: 下载 GIF（Download GIF）

### 4.3 转换为 PXL

```bash
python pixel_tool.py convert blink.gif -o blink.pxl
```

指定参数：

```bash
# 限制最大帧数为 8，帧间隔 150ms
python pixel_tool.py convert blink.gif --max-frames 8 -i 150 -o blink.pxl

# 不循环播放
python pixel_tool.py convert intro.gif --no-loop -o intro.pxl
```

### 4.4 帧间隔处理

- PXL 编码器会自动读取 GIF 的帧间隔信息
- 若 GIF 未设置帧间隔，默认使用 100ms
- 最小帧间隔限制为 50ms，低于此值会被强制调整
- 可通过 `-i` 参数覆盖原始帧间隔

### 4.5 帧数控制

GIF 可能包含大量帧（如 30+ 帧），建议使用 `--max-frames` 限制：

- 32x32 分辨率下，16 帧以内已足够表达大多数动画
- 帧数越多，文件越大，传输时间越长
- Delta 压缩可有效减小后续帧的体积

## 五、Spritesheet 使用方法

### 5.1 什么是 Spritesheet

Spritesheet（雪碧图）是将多帧动画水平排列在一张图片中的格式，广泛用于游戏开发和像素动画制作。

### 5.2 制作要求

- 所有帧必须等宽等高，水平排列
- 帧之间无缝隙
- 例如 4 帧 32x32 动画：总图片宽度 = 128 像素，高度 = 32 像素

```
|  帧0  |  帧1  |  帧2  |  帧3  |
| 32x32 | 32x32 | 32x32 | 32x32 |
```

### 5.3 Aseprite 导出 Spritesheet

1. File > Export Spritesheet
2. Layout: Horizontal Strip（水平排列）
3. 输出格式: PNG
4. 点击 Export

### 5.4 Piskel 导出 Spritesheet

1. 点击下载按钮旁的下拉箭头
2. 选择 "Download as PNG (Spritesheet)"
3. 布局自动为水平排列

### 5.5 转换为 PXL

```bash
# 使用 --sprite 标志启用雪碧图模式
python pixel_tool.py convert walk_sprite.png --sprite -o walk.pxl

# 指定帧间隔
python pixel_tool.py convert attack_sprite.png --sprite -i 100 -o attack.pxl
```

**自动帧数检测**：编码器通过 `图片宽度 / 图片高度` 自动计算帧数。例如 128x32 的雪碧图会被识别为 4 帧。

### 5.6 注意事项

- 雪碧图必须是正方形帧的水平排列，编码器以高度作为单帧宽度进行裁切
- 若图片宽度 <= 高度，将被当作单张静态图片处理
- 帧数 = `width // height`，不足一帧的部分会被忽略

## 六、压缩模式选择建议

PXL 格式支持三种像素存储模式，根据使用场景选择最优方案：

### 6.1 无压缩（原始 RGB565）

- **适用**：静态图片、小文件
- **特点**：编码/解码最快，文件大小固定
- **大小**：`width * height * 2` 字节/帧（32x32 = 2048 字节/帧）
- **示例**：图标、表情、Logo

### 6.2 RLE 压缩

- **适用**：大面积纯色的静态图片
- **特点**：对纯色区域压缩效果好，对复杂图像效果有限
- **压缩率**：纯色背景图片可达 30-50%，复杂图片几乎无压缩
- **注意**：当前 `convert` 命令默认不启用 RLE，需要通过编程接口手动设置 flags

### 6.3 Delta 差分帧压缩（推荐用于动画）

- **适用**：多帧动画（GIF、Spritesheet）
- **特点**：仅记录帧间变化的像素，动画越连贯压缩效果越好
- **压缩率**：通常 30-70%（即减少 30-70% 数据量），局部变化大的动画效果更佳
- **自动启用**：`convert` 命令检测到多帧时自动使用 Delta 编码

### 6.4 选择决策树

```
是静态图片吗？
├── 是 → 使用无压缩（默认）
│        大面积纯色？→ 可考虑 RLE（编程接口）
└── 否（多帧动画）→ 使用 Delta 压缩（自动启用）
```

### 6.5 压缩效果对比

以一个 12 帧 32x32 动画为例：

| 模式 | 文件大小 | 说明 |
|------|----------|------|
| 无压缩 | 24,576 + 16 = 24,592 bytes | 每帧完整存储 |
| Delta | 约 5,000-10,000 bytes | 仅存差异像素 |

- **设计要点**：心形使用对称绘制，线条宽度 2 像素，红色 `#FF0000`

- **类型**：静态图标
- **内容**：黄色笑脸，圆脸 + 眼睛 + 嘴巴
- **转换命令**：

```bash
python pixel_tool.py convert examples/smile.png -o examples/smile.pxl
```

- **发送到设备**：

```bash
python pixel_tool.py send examples/smile.pxl 192.168.1.100
```

- **设计要点**：使用 2-3 种黄色系（深黄轮廓 + 浅黄填充），眼睛和嘴巴留出足够间距

### 7.3 rainbow -- 彩虹条纹

- **类型**：静态图标
- **内容**：水平彩虹条纹
- **转换命令**：


### 7.4 完整工作流示例

以下是一个从零开始制作心形图标并推送到设备的完整流程：

```bash
# 1. 用绘图工具制作 32x32 的 heart.png

# 2. 转换为 PXL
python pixel_tool.py convert heart.png -o heart.pxl

python pixel_tool.py send heart.pxl 192.168.1.100

python pixel_tool.py cmd stop 192.168.1.100
```

## 八、优化技巧

### 8.1 减少颜色数量

- 在绘图工具中使用限制调色板（如 16 色或 32 色）
- 更少的颜色意味着更多连续相同像素，RLE 压缩效果更好
- 在 Aseprite 中：Sprite > Color Mode > Indexed

### 8.2 减少帧数

- 评估动画流畅度与文件大小的平衡
- 4 帧 200ms 的循环动画已能表达大多数动作
- 使用洋葱皮检查相邻帧的差异，合并过于接近的帧

### 8.3 利用帧间相似性

- 动画中尽量只变化必要的部分，保持背景不变
- 例如眨眼动画：仅眼睛区域变化，其余保持不动
- 这样 Delta 压缩只编码眼睛区域的少量像素，文件极小

### 8.4 合理设置帧间隔

- 帧间隔过短（< 80ms）：ESP32 可能来不及处理，导致丢帧
- 帧间隔过长（> 500ms）：动画看起来卡顿
- 推荐范围：100-200ms，根据动画速度调整

### 8.5 文件大小预估

静态图：

```
大小 = 16 (头) + width * height * 2 (像素)
32x32 静态 = 16 + 2048 = 2064 bytes
```

Delta 压缩动画：

```
大小 = 16 (头) + 2048 (首帧) + 压缩后续帧
实际大小取决于帧间变化量，可通过 `info` 命令查看
```

### 8.6 传输优化

- WiFi 信号弱时，可增大 `-t` 超时参数
- 大文件传输期间避免频繁的 WiFi 流量操作
- 若传输中断，重新发送即可，ESP32 会覆盖之前接收的不完整数据

### 8.7 调试技巧

- 先用 `info` 检查 PXL 文件是否正确
- 在 PC 上用 Python 预览图片效果后再发送
- 发送后若画面异常，用 `cmd stop` 停止播放，检查源图片是否正确

### 8.8 常见动画类型参考

| 动画类型 | 帧数 | 帧间隔 | 技巧 |
|----------|------|--------|------|
| 呼吸/脉动 | 2 | 400ms | 仅改变亮度或大小 |
| 眨眼 | 2-3 | 200ms | 仅眼睛区域变化 |
| 点头 | 2-4 | 200ms | 头部区域上下偏移 1-2 像素 |
| 走路 | 4-8 | 120ms | 腿部循环，上半身相对稳定 |
| 跳舞 | 8-16 | 100ms | 全身动作，帧间变化较大 |
| 消失/出现 | 4-8 | 150ms | 逐像素淡入淡出或闪烁 |
