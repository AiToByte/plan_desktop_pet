# 硬件组件选型详解

> 本文档详细介绍桌面电子宠物项目中所有硬件组件的选型依据、电气参数、接线方式和驱动原理。
> 基于微雪 ESP32-S3 1.54inch LCD 开发板，外接 BH1750 光照传感器和 DRV2605L 触觉驱动器。

## 二、主控: ESP32-S3-WROOM-1

### 2.1 核心规格

| 参数 | 规格 |
|------|------|
| CPU | Xtensa LX7 双核 @ 240MHz |
| SRAM | 512KB |
| PSRAM | 8MB (Octal SPI, OPI模式) |
| Flash | 16MB (Quad SPI) |
| WiFi | 802.11 b/g/n 2.4GHz |
| 蓝牙 | BLE 5.0 |
| USB | USB-OTG (Type-C) |
| GPIO | 45个可用GPIO |
| ADC | 2x 12-bit SAR ADC (20通道) |
| 工作电压 | 2.3V ~ 3.6V |
| 封装 | QFN56 (7x7mm) |
| 工作温度 | -40°C ~ +85°C |

### 2.2 双核任务分配

固件采用 FreeRTOS 双核架构，充分利用 ESP32-S3 的双核能力：

```
Core 0 (通信核心)                    Core 1 (渲染核心)
┌─────────────────────┐            ┌─────────────────────┐
│ WiFi 管理            │            │ LCD 显示渲染         │
│ TCP 客户端通信       │            │ 动画帧更新           │
│ BLE 配置服务        │            │ Sprite 绘制          │
│ 传感器数据采集       │            │ 像素表情播放         │
│ 心跳/重连逻辑       │            │ 思考链滚动展示       │
│ 栈: 8192 bytes      │            │ 栈: 16384 bytes     │
└─────────────────────┘            └─────────────────────┘
```

**配置宏定义：**
```cpp
#define COMM_TASK_CORE    0       // 通信任务运行在Core 0
#define RENDER_TASK_CORE  1       // 渲染任务运行在Core 1
#define COMM_TASK_STACK   8192    // 通信任务栈大小
#define RENDER_TASK_STACK 16384   // 渲染任务栈较大(含sprite操作)
```

### 2.3 内存分配策略

| 内存区域 | 大小 | 用途 |
|---------|------|------|
| SRAM (内部) | 512KB | 代码段、栈、FreeRTOS堆、DMA缓冲 |
| PSRAM (OPI) | 8MB | Sprite帧缓冲、思考链历史环形缓冲(40步)、像素动画缓存 |
| Flash (QPI) | 16MB | 固件代码、字体资源、像素动画数据 |

**PSRAM 关键用途：**
- LCD 帧缓冲: 240x240x2 = 112.5KB (RGB565)
- 思考链历史: 40步 x (4B时间戳 + 64B文本) = 2.7KB
- 双缓冲 Sprite: 2 x 112.5KB = 225KB

### 2.4 WiFi 省电策略

固件实现了三级省电模式，优化日常功耗：

| 模式 | 值 | 说明 | 功耗 |
|------|---|------|------|
| ACTIVE | 0 | 全速运行 | ~80mA |
| MODEM_SLEEP_AUTO | 1 | 自动休眠WiFi射频 | ~15mA |
| LIGHT_SLEEP | 2 | 轻度休眠（推荐） | ~5mA |

**省电配置参数：**
```cpp
#define IDLE_POWER_MODE        2     // 空闲省电模式 (LIGHT_SLEEP)
#define DTIM_ACTIVE            1     // 活跃态DTIM间隔
#define DTIM_IDLE              10    // 空闲态DTIM间隔
#define IDLE_TIMEOUT_MS        30000 // 进入省电模式的空闲超时 30秒
#define WAKE_CHECK_INTERVAL_MS 500   // 唤醒检查间隔 500ms
#define BSS_MAX_IDLE_SEC       300   // BSS最大空闲时间 5分钟
```

### 2.5 为什么选 ESP32-S3

| 对比维度 | ESP32 | ESP32-S2 | ESP32-C3 | **ESP32-S3** |
|---------|-------|----------|----------|-------------|
| CPU核心 | 双核LX6 | 单核LX7 | 单核RISC-V | **双核LX7** |
| 主频 | 240MHz | 240MHz | 160MHz | **240MHz** |
| PSRAM | 4MB (SPI) | 2MB (QPI) | 无 | **8MB (OPI)** |
| 蓝牙 | BT4.2 | 无 | BLE5.0 | **BLE5.0** |
| USB-OTG | 无 | 有 | 无 | **有** |
| 触摸通道 | 10 | 14 | 0 | **14** |
| AI加速 | 无 | 无 | 无 | **向量指令** |
| 价格 | ~15元 | ~12元 | ~8元 | **~18元** |

**选择 ESP32-S3 的理由：**

1. **双核架构**: 通信与渲染分离，避免WiFi收发阻塞动画刷新
2. **8MB OPI PSRAM**: 足够存放双缓冲Sprite和大量动画帧数据
3. **BLE 5.0**: 支持低功耗蓝牙配网和数据传输
4. **USB-OTG**: Type-C直连烧录，无需额外UART转换器
5. **14通道触摸**: 支持电容触摸按键和接近感应
6. **向量指令**: 未来可扩展像素图像处理加速

### 2.6 晶振与启动

- **外部晶振**: 40MHz (3215封装)
- **启动模式**: GPIO0 低电平进入下载模式，高电平正常启动
- **上电时序**: EN引脚内部RC延时，VDD稳定后自动启动

### 3.3 SPI 通信配置

**LovyanGFX 驱动配置（display_manager.h）：**

```cpp
// SPI总线配置
cfg.spi_host   = SPI2_HOST;      // 使用SPI2主机
cfg.spi_mode   = 0;              // SPI模式0 (CPOL=0, CPHA=0)
cfg.freq_write = 80000000;       // 写入频率 80MHz
cfg.freq_read  = 16000000;       // 读取频率 16MHz
cfg.pin_sclk   = LCD_SCLK;      // GPIO12
cfg.pin_mosi   = LCD_MOSI;      // GPIO11
cfg.pin_miso   = LCD_MISO;      // GPIO13
cfg.dma_channel = SPI_DMA_CH_AUTO; // DMA自动分配通道

// 面板配置
pcfg.memory_width  = 240;
pcfg.memory_height = 240;
pcfg.panel_width   = 240;
pcfg.panel_height  = 240;
pcfg.invert        = true;       // 颜色反转 (IPS屏常见)
pcfg.rgb_order     = false;      // BGR顺序

// 背光PWM配置
lcfg.pin_bl    = LCD_BL;         // GPIO48
lcfg.freq      = 44100;          // PWM频率 44.1kHz
lcfg.pwm_channel = 7;            // PWM通道7
lcfg.invert    = false;          // 不反转
```

**SPI 时序说明：**

```
        ┌───┐   ┌───┐   ┌───┐   ┌───┐
SCLK ───┘   └───┘   └───┘   └───┘   └───
        ┌───────────────────────────────
MOSI ───┤ D7 │ D6 │ D5 │ ... │ D0 │
        └───────────────────────────────
          │                            │
CS   ─────┘                            └─────
          │◄── 8-bit 数据传输 ──────►│

DC=0: 命令模式 (MOSI传输的是命令字节)
DC=1: 数据模式 (MOSI传输的是像素数据)
```

### 3.4 TE 引脚 (V-Sync)

ST7789V 提供 Tearing Effect (TE) 输出引脚，用于同步 MCU 写入与 LCD 扫描：

```cpp
#define LCD_TE_PIN        -1      // -1=禁用(当前使用软件帧率控制)
#define TE_ACTIVE_HIGH    true    // TE信号极性: 上升沿有效
#define VSYNC_TIMEOUT_MS  20      // 等待V-Sync超时 20ms
```

**TE 信号时序：**
```
LCD扫描 ──────────────────────────────────────►
         │ 帧N显示中 │ 帧N+1准备 │ 帧N+1显示中 │
         │           │           │             │
TE信号 ──┘           └───────────┘             └──
                    ▲
                    │ MCU在此时刻开始写入下一帧
                    │ 避免画面撕裂
```

当前固件使用软件帧率控制，TE引脚预留为 -1（禁用）。硬件接线确认后，将 `LCD_TE_PIN` 改为实际 GPIO 编号即可启用硬件 V-Sync。

### 3.5 RGB565 色彩格式详解

ST7789V 使用 RGB565 格式，每个像素占 2 字节（16-bit）：

```
高字节 (Byte 0)          低字节 (Byte 1)
┌─┬─┬─┬─┬─┬─┬─┬─┐    ┌─┬─┬─┬─┬─┬─┬─┬─┐
│R4│R3│R2│R1│R0│G5│G4│G3││G2│G1│G0│B4│B3│B2│B1│B0│
└─┴─┴─┴─┴─┴─┴─┴─┘    └─┴─┴─┴─┴─┴─┴─┴─┘
  5-bit 红色              6-bit 绿色    5-bit 蓝色
```

| 颜色分量 | 位数 | 范围 | 说明 |
|---------|------|------|------|
| Red | 5-bit | 0~31 | 红色通道 |
| Green | 6-bit | 0~63 | 绿色通道（精度最高，人眼敏感） |
| Blue | 5-bit | 0~31 | 蓝色通道 |

**帧缓冲大小计算：**
```
单帧 = 240 x 240 x 2 bytes = 115,200 bytes = 112.5 KB
双缓冲 = 2 x 112.5 KB = 225 KB
```

### 3.6 背光控制

背光通过 PWM 调节亮度，占空比 0%~100% 对应最暗到最亮：

```cpp
#define LCD_BRIGHTNESS  200    // 默认亮度 (0-255)
// PWM通道: 7, 频率: 44.1kHz
```

**背光驱动电路：**
```
GPIO48 ──[R 100Ω]──► LCD BL+
                      LCD BL- ──► GND

背光LED参数:
  典型电流: 20mA
  最大电流: 30mA
  限流电阻: 100Ω (5V供电时, I = (5-3.2)/100 ≈ 18mA)
```

**自动调光映射曲线（基于BH1750光照值）：**

| 环境光照 (lux) | 背光亮度 (%) | PWM值 |
|---------------|-------------|-------|
| < 10 | 20% | 51 |
| 10 ~ 100 | 20% ~ 50% | 51 ~ 128 |
| 100 ~ 1000 | 50% ~ 100% | 128 ~ 255 |
| > 1000 | 100% | 255 |

### 4.3 工作模式

BH1750 支持多种测量模式，本项目使用 **连续高分辨率模式**：

| 模式 | 指令 | 分辨率 | 测量时间 | 精度 | 说明 |
|------|------|--------|---------|------|------|

| 连续高分辨率2 | 0x11 | 0.5 lux | 120~180ms | ±20% | 精度翻倍 |
| 连续低分辨率 | 0x13 | 4 lux | 16ms | ±20% | 快速测量 |
| 单次高分辨率 | 0x20 | 1 lux | 120~180ms | ±20% | 测完自动断电 |
| 单次低分辨率 | 0x23 | 4 lux | 16ms | ±20% | 快速单次 |

**固件指令定义：**
```cpp

static constexpr uint8_t BH1750_RESET     = 0x07;  // 重置

```

### 4.4 I2C 通信时序

```
主机(ESP32)                           从机(BH1750)
    │                                      │
    │──── START ──────────────────────────►│
    │──── [0x46] (写地址) ────────────────►│  0x23<<1 | W
    │◄───── ACK ──────────────────────────│
    │──── [0x10] (连续高分辨率) ──────────►│
    │◄───── ACK ──────────────────────────│
    │                                      │
    │  ... 等待 120~180ms 测量完成 ...      │
    │                                      │
    │──── START ──────────────────────────►│
    │──── [0x47] (读地址) ────────────────►│  0x23<<1 | R
    │◄───── ACK ──────────────────────────│
    │◄───── [高字节] ────────────────────│
    │──── ACK ────────────────────────────►│
    │◄───── [低字节] ────────────────────│
    │──── NACK ────────────────────────────►│
    │──── STOP ────────────────────────────►│

    原始值 = (高字节 << 8) | 低字节
    实际lux = 原始值 / 1.2
```

### 4.5 EMA 滤波算法

为避免光照读数抖动，固件实现了指数移动平均（EMA）滤波：

```cpp
float _emaLux;                      // EMA滤波后的lux值
static constexpr float EMA_ALPHA = 0.15f;  // 平滑系数

// 滤波公式: ema = alpha * new_value + (1 - alpha) * ema
_emaLux = EMA_ALPHA * rawLux + (1.0f - EMA_ALPHA) * _emaLux;
```

**EMA 系数选择：**

| alpha值 | 响应速度 | 平滑度 | 适用场景 |
|---------|---------|--------|---------|
| 0.05 | 很慢 | 非常平滑 | 长期趋势 |
| 0.15 | 中等 | 较平滑 | **本项目使用** |
| 0.30 | 较快 | 轻微抖动 | 快速响应 |
| 1.00 | 即时 | 无滤波 | 原始值 |

## 五、触觉驱动: DRV2605L

### 5.1 驱动器规格

| 参数 | 规格 |

| 型号 | DRV2605L (TI) |

| 内置效果库 | 123种预设效果 |
| 波形序列器 | 8个槽位 |

| 模块封装 | GY-DRV2605L Breakout Board |

### 5.2 接线方式

```
ESP32-S3              DRV2605L 模块
────────              ─────────────
GPIO41 (SDA) ──────── SDA
GPIO42 (SCL) ──────── SCL
3.3V         ──────── VDD
GND          ──────── GND
                       OUT+ ──► LRA马达正极
                       OUT- ──► LRA马达负极
```

**注意: DRV2605L 与 BH1750 共享 I2C 总线（详见第八章）。**

### 5.3 效果库详解

DRV2605L 内置 123 种触觉效果，分为两大库：

- **Library A (ERM)**: 偏心旋转马达效果 (效果 1~123)
- **Library B (LRA)**: 线性谐振马达效果 (效果 1~123)
- **Library LRA (Library 6)**: 专为 LRA 优化的效果集

**本项目预定义效果：**

| 效果名称 | ID | 类型 | 用途 | 描述 |
|---------|----|------|------|------|
| Click | 1 | 轻击 | 触摸确认反馈 | 短促清晰的点击感 |
| Soft | 3 | 软触 | 滑动反馈 | 柔和的触感 |
| Buzz | 47 | 嗡嗡 | 通知提醒 | 持续振动 |
| Strong | 72 | 强击 | 警告提示 | 强烈的冲击感 |
| Ramp Up | 12 | 渐强 | 情绪表达 | 由弱渐强的振动 |

**固件快捷方法：**
```cpp
void click();      // 轻击反馈 (触摸确认)
void buzz();       // 持续嗡嗡 (通知)
void strongHit();  // 强击 (警告)
void softTouch();  // 软触 (滑动反馈)
```

### 5.4 波形序列器

DRV2605L 提供 8 个波形槽位，可组合播放复杂触觉模式：

```
槽位:  [0]   [1]   [2]   [3]   [4]   [5]   [6]   [7]
       效果1  效果2  效果3  效果4  0x00  0x00  0x00  0x00
                                      ↑ 空槽(停止标记)

播放顺序: [0] → [1] → [2] → [3] → 停止
```

**波形序列示例：**
```cpp
// 播放"心跳"触觉: 软触 → 暂停 → 软触
uint8_t heartbeat[] = {3, 0, 3};  // 效果3(软触), 空槽(间隔), 效果3(软触)
playWaveform(heartbeat, 3);
```

### 5.5 自动校准流程

DRV2605L 上电后需对 LRA 马达进行自动校准，以获得最佳驱动效果：

```
校准流程:
1. 设置控制寄存器 (Control1~Control5)
2. 设置额定电压: 1.8V RMS
3. 设置过驱钳位电压: 2.5V
4. 选择马达类型: LRA (Library 6)
5. 触发自动校准 (GO位)
6. 等待校准完成 (GO位清零)
7. 读取校准结果 (反馈补偿值)
```

**关键寄存器配置：**

| 寄存器 | 地址 | 值 | 说明 |
|--------|------|----|------|
| Mode | 0x01 | 0x00 | 正常模式 |
| Rated Voltage | 0x16 | 0x53 | LRA额定电压 1.8Vrms |
| OD Clamp | 0x17 | 0x88 | 过驱钳位 2.5V |
| Control1 | 0x1E | 0x93 | 启动/制动时间 |
| Control2 | 0x1F | 0xF5 | 采样时间/BLANK |
| Control3 | 0x20 | 0x80 | LRA模式/ERM库选择 |
| Control4 | 0x21 | 0x30 | 自动校准配置 |
| Feedback | 0x3A | 0xB6 | 反馈控制 |

### 5.6 LRA 与 ERM 对比

| 特性 | LRA (线性谐振) | ERM (偏心旋转) |
|------|---------------|---------------|
| 振动方式 | 线性往复 | 旋转偏心 |
| 响应速度 | 快 (~5ms) | 慢 (~30ms) |
| 精确度 | 高 | 低 |
| 功耗 | 低 | 中 |
| 噪音 | 低 | 较高 |
| 效果表现 | 清脆、精准 | 浑厚、持续 |
| 价格 | ~3元 | ~1.5元 |
| 本项目 | **使用** | 备选 |

## 七、触摸: ESP32 Touch 外设

### 7.1 触摸通道规格

| 参数 | 规格 |
|------|------|
| 引脚 | GPIO1 (Touch1 通道) |
| 感应方式 | 电容式触摸 |
| 检测原理 | 电容充放电时间测量 |
| 分辨率 | 16-bit (0~65535) |
| 阈值 | 40 (需校准) |
| 手势支持 | 单击/双击/长按/接近 |

### 7.2 电容触摸原理

ESP32 内部 Touch 外设通过测量引脚电容变化来检测触摸：

```
无触摸时:
  GPIO1 ──[Cp 约20pF]── GND
           │
           └── 芯片内部: 充放电周期较长, 读数高 (约200~400)

手指触摸时:
  GPIO1 ──[Cp + Cfinger 约5~20pF]── GND
           │           │
           └── 芯片内部: 充放电周期变短, 读数降低 (约100~200)
                         差值 = 基线 - 当前值 > 阈值 → 判定为触摸
```

**检测公式：**
```
触摸判定: (基线值 - 当前值) > 阈值
其中:
  基线值 = 校准时的无触摸读数 (20次采样平均)
  当前值 = 实时采样读数
  阈值   = 40 (灵敏度, 可调整)
```

### 7.3 校准方法

固件在初始化时自动进行触摸校准：

```
校准流程:
1. 采集 20 个无触摸样本
2. 去除最高 4 个和最低 4 个 (抗干扰)
3. 对剩余 12 个取平均值作为基线
4. 设置阈值 = 40 (可灵敏度调整)

代码流程:
  for (i = 0; i < 20; i++) {
      samples[i] = touchRead(TOUCH_PIN);
      delay(10);
  }
  sort(samples);
  baseline = average(samples[4..15]);  // 去掉极值
```

### 7.4 手势识别

固件实现了四种触摸手势识别：

```
时间轴: ──────────────────────────────────────────────────►

单击 (Single Tap):
  │按下│释放│
  │◄─┤    │  < 400ms 且无第二次按下

双击 (Double Tap):
  │按下│释放│  │按下│释放│
  │◄─┤    ├──┤◄─┤    │  两次间隔 < 400ms

长按 (Long Press):
  │按下│──────────────│释放│
  │◄── 持续 ≥ 1000ms ──►│

接近 (Proximity):
  │──────── 持续感应 ────────│
  │  EMA快线 - EMA慢线 > 阈值  │  手指靠近但未接触
```

**手势配置参数：**
```cpp
#define TOUCH_THRESHOLD      40      // 触摸阈值
#define TOUCH_LONG_PRESS_MS  1000    // 长按判定时间 1秒
// 双击间隔: 400ms (代码中定义)
```

### 7.5 接近感应 (Proximity)

固件利用同一触摸引脚实现了接近感应功能，通过 EMA 快慢线差分检测手指靠近：

```cpp
// EMA 快慢线配置
#define PROX_EMA_FAST_ALPHA    0.3f   // 快速EMA系数 (响应快)
#define PROX_EMA_SLOW_ALPHA    0.05f  // 慢速EMA系数 (基线跟踪)
#define PROX_RISING_THRESHOLD  8      // 上升差分阈值 (接近检测)
#define PROX_FALLING_THRESHOLD 4      // 下降差分阈值 (远离检测)
#define PROX_COOLDOWN_MS       2000   // 接近事件冷却时间 2秒
#define PROX_WAKE_DURATION_MS  15000  // 接近唤醒后亮屏时长 15秒
```

**接近检测算法：**
```
快速EMA: ema_fast = 0.3 * raw + 0.7 * ema_fast  (响应快)
慢速EMA: ema_slow = 0.05 * raw + 0.95 * ema_slow (基线跟踪)

差分: diff = ema_fast - ema_slow

判断:
  diff > +8  → 手指接近 (上升沿触发)
  diff < -4  → 手指远离 (下降沿触发)

冷却: 两次接近事件间隔 ≥ 2秒 (防抖)

应用场景: 手指靠近屏幕 → 自动唤醒背光 (15秒后自动熄灭)
```

## 八、I2C 总线共享

### 8.1 总线拓扑

BH1750 光照传感器和 DRV2605L 触觉驱动器共享同一条 I2C 总线：

```
                    I2C 总线 (SDA=GPIO41, SCL=GPIO42)
                    ══════════════════════════════════
                           │                 │
                    ┌──────┴──────┐   ┌──────┴──────┐
                    │   BH1750    │   │  DRV2605L   │
                    │  地址: 0x23 │   │  地址: 0x5A │
                    │   光照传感器 │   │  触觉驱动器  │
                    └─────────────┘   └─────────────┘

    GPIO41 (SDA) ───┬─────────────────────────────────┬─── 上拉电阻(4.7kΩ) ── 3.3V
    GPIO42 (SCL) ───┴─────────────────────────────────┴─── 上拉电阻(4.7kΩ) ── 3.3V
```

### 8.2 地址冲突分析

| 设备 | 7-bit地址 | 8-bit写地址 | 8-bit读地址 | 冲突 |
|------|----------|-----------|-----------|------|
| BH1750 | 0x23 | 0x46 | 0x47 | 无 |
| DRV2605L | 0x5A | 0xB4 | 0xB5 | 无 |

**结论: 两个设备地址完全不同 (0x23 vs 0x5A)，不存在地址冲突。**

### 8.3 上拉电阻需求

I2C 总线需要上拉电阻确保空闲时为高电平：

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 上拉电阻 | 4.7kΩ | 标准I2C上拉 |
| 总线电容 | < 400pF | 含走线+器件寄生电容 |
| 上升时间 | < 300ns (100kHz) / < 30ns (400kHz) | I2C规范 |
| 总线速率 | 400kHz (Fast Mode) | 本项目使用 |

**上拉电阻计算：**
```
R_pullup = (VDD - V_OL) / I_OL
         = (3.3V - 0.4V) / 3mA
         = 967Ω (最小值)

推荐: 4.7kΩ (标准值, 兼顾功耗和速度)
高速: 2.2kΩ (400kHz时上升沿更快)
```

**注意:** 大多数 BH1750 和 DRV2605L 模块板上已集成上拉电阻，外接时需确认是否需要移除板载上拉以避免并联导致阻值过低。

### 8.4 时序仲裁

I2C 总线是多主多从的半双工协议，共享总线时需注意：

```
通信时序示意:

时间 ──────────────────────────────────────────────────────►

ESP32 (主机)
  │── 读取BH1750 ──│── 读取BH1750完成 ──│── 触发DRV2605L ──│
  │  (120~180ms)   │                    │  (几ms)          │
  │                │                    │                   │
  读取间隔: 2000ms (BH1750_READ_INTERVAL)
```

**总线冲突避免策略：**

1. **主从模式**: ESP32 始终作为唯一主机，不会发生多主机冲突
2. **顺序访问**: 固件在同一时刻只与一个从机通信
3. **读取间隔**: BH1750 每 2 秒读取一次，DRV2605L 触发式调用
4. **DMA缓冲**: I2C 事务使用 DMA，不阻塞 CPU

**固件隔离保障：**
```cpp
// BH1750 和 DRV2605L 各自持有独立的 TwoWire 实例
TwoWire _wire;  // AmbientLightManager 中的 I2C 实例
TwoWire _wire;  // HapticDriver 中的 I2C 实例
// 实际共享物理总线，但软件层面隔离
```

### 9.2 启动模式选择

ESP32-S3 上电时根据 GPIO0 电平选择启动模式：

| GPIO0电平 | 启动模式 | 说明 |
|----------|---------|------|
| 高 (默认) | Flash启动 | 正常运行固件 |
| 低 (按下) | ROM下载模式 | 通过USB烧录固件 |

**启动时序：**
```
EN引脚 ──┘──────────────────────── (释放复位)
GPIO0 ────────┐                    (采样启动模式)
              │
              ▼
         ┌──────────┐
         │ 采样GPIO0 │
         └────┬─────┘
              │
        ┌─────┴─────┐
        │ GPIO0=?   │
        ├──────┬────┤
        │ HIGH │LOW │
        ▼      ▼    │
     Flash启动  ROM下载
```

### 9.3 运行时功能

固件在正常运行时复用 BOOT 按钮实现额外功能：

- **Web 配网触发**: 长按 BOOT 按钮进入配网模式 (AP热点: `Pet-Setup`)
- **固件恢复**: 双击 BOOT 按钮恢复出厂设置
- **调试模式**: 特定按键组合进入串口调试

### 9.4 按键防抖

```
GPIO0 ──[R 10kΩ]── 3.3V    (外部上拉, 提高抗干扰)
     │
     └──[按键]── GND
     │
     └──[PESD5V0S1BA]── GND  (ESD保护, 可选)
```

- 内部上拉: ~45kΩ (ESP32-S3内置)
- 外部上拉: 10kΩ (推荐, 提高抗干扰能力)
- 软件防抖: 50ms 消抖延时
- ESD保护: PESD5V0S1BA 二极管 (可选)

### 10.3 采购注意事项

1. **确认开发板版本**: 购买前确认是"1.54inch"版本，非"1.28inch圆屏"版本
2. **PSRAM 容量**: 选择带 8MB PSRAM 的版本（动画缓存必需）
3. **BH1750 模块**: 选择已焊接排针的版本，确认 ADDR 引脚接 GND (地址 0x23)
4. **DRV2605L 模块**: 确认附带 LRA 马达，或单独采购 10mm 圆形 LRA
5. **数据线**: 确认 Type-C 线支持数据传输，非纯充电线
6. **蜂鸣器**: 必须选择**无源**蜂鸣器，有源蜂鸣器无法控制音调

### 10.4 开发板引脚映射差异

不同厂家的 ESP32-S3 开发板引脚映射可能不同，以下是本项目使用的微雪板引脚：

| 功能 | 微雪板 (本项目) | 常见替代映射 |
|------|----------------|-------------|
| LCD CS | GPIO5 | GPIO10 |
| LCD DC | GPIO2 | GPIO13 |
| LCD RST | GPIO4 | GPIO14 |
| LCD MOSI | GPIO11 | GPIO6 |
| LCD SCLK | GPIO12 | GPIO7 |
| LCD BL | GPIO48 | GPIO21 |

**使用其他开发板时，需修改 `config.h` 中的引脚定义。**
