## 1. 产品简介

### 这是什么？

桌面电子宠物是一款基于 ESP32-S3 的智能桌面伴侣设备。它通过 1.54 英寸彩色 LCD 屏幕显示像素风格的表情动画，能够实时展示天气信息、AI 助手的工作状态和资源使用情况，并通过触摸、旋钮、声音和振动与您互动。

### 核心功能一览

| 功能 | 说明 |
|------|------|
| 🎭 **表情动画** | 4 种情绪状态，像素风格动态表情 |
| 🌤️ **天气显示** | 实时天气（温度、体感温度、湿度、风速、天气图标） |
| 📊 **状态监控** | AI 助手的 Token 用量、CPU / 内存使用率 |
| 👆 **触摸交互** | 单击 / 双击 / 长按 / 接近感应，4 种手势 |
| 🎛️ **旋钮控制** | 旋转调节 + 按压确认 |
| 🔊 **声音提示** | 启动音、通知音、警报音等多种音效 |
| 📳 **触觉反馈** | 点击、嗡嗡、强击等振动反馈 |
| 💡 **自动调光** | 环境光感应，自动调节屏幕亮度 |
| 🌙 **夜览模式** | 夜间自动暖色调，保护眼睛 |
| 🔋 **智能省电** | 自动变暗 → 屏幕休眠 → 深度休眠三级节能 |
| 📡 **OTA 升级** | 无线固件更新，无需拆机 |

## 3. 首次使用：快速上手

### 3.1 硬件连接

```
步骤 1: 将蜂鸣器模块连接到主板蜂鸣器接口
步骤 2: 将 BH1750 光照传感器 + DRV2605L 振动马达连接到 I2C 总线（SDA=GPIO41, SCL=GPIO42）
步骤 3: 将旋转编码器连接到编码器接口（A=GPIO47, B=GPIO48, SW=GPIO21）
步骤 4: 用 USB-C 线连接设备到电脑或电源适配器
```

> 💡 **提示**: 所有外接模块均为即插即用，无需焊接。连接后设备会自动检测模块是否存在，缺少某个模块不影响其他功能正常使用。

### 3.2 首次开机

接通电源后，设备将：

1. **屏幕亮起**，显示启动画面
2. **播放启动音效**（"嘟嘟"两声）
3. **振动一下**，确认触觉模块就绪
4. 显示 **"Connecting WiFi..."**

如果 WiFi 未配置，设备会自动进入**配网模式**（详见下方）。

### 3.3 配网（连接 WiFi）

设备支持 **3 种配网方式**，任选其一：

---

#### 方式一：Web 配网（推荐 ⭐）

最简单的方式，用手机或电脑浏览器操作。

```
┌─────────────────────────────────────────────────────┐
│  设备开机 → 未找到已保存WiFi → 自动进入配网模式      │
│                                                      │
│  ① 手机/电脑搜索WiFi → 连接 "Pet-Setup"             │
│     密码: 12345678                                   │
│                                                      │
│  ② 浏览器自动弹出配网页面                            │
│     (如未弹出，手动访问 http://192.168.4.1)          │
│                                                      │
│  ③ 选择您的家庭WiFi → 输入密码 → 点击"连接"         │
│                                                      │
│  ④ 设备自动重启并连接WiFi ✅                         │
└─────────────────────────────────────────────────────┘
```

**详细步骤**：

1. **确认设备处于配网模式**
   - 屏幕显示 "WiFi Setup" 或类似提示
   - 设备已创建名为 **`Pet-Setup`** 的 WiFi 热点

2. **连接热点**
   - 用手机或电脑搜索 WiFi，找到 **`Pet-Setup`**
   - 密码：**`12345678`**

3. **打开配网页面**
   - 连接后浏览器通常会自动弹出配网页面
   - 如未弹出，请手动在浏览器地址栏输入：**`http://192.168.4.1`**

4. **选择 WiFi 并输入密码**
   - 页面会列出扫描到的 WiFi 列表
   - 点击您的家庭 WiFi，输入密码
   - 点击 **"连接"** 按钮

5. **等待连接**
   - 设备会自动保存 WiFi 信息并重启
   - 重启后自动连接到您指定的 WiFi

> ⏱️ **注意**: 配网超时时间为 **2 分钟**。如超时未完成，设备会自动重启进入配网模式，请重新操作。

---

#### 方式二：BLE 蓝牙配网

适用于手机端 App 配网（需配套 App 支持）。

```
① 设备进入配网模式后，同时开启 BLE 广播
② 手机 App 搜索蓝牙设备 → 连接 "Pet-Setup"
③ 通过蓝牙发送 WiFi 名称和密码
④ 设备保存并自动连接
```

---

#### 方式三：UDP 自动发现

适用于局域网内 PC 端软件自动配网。

```
① PC 端监控软件运行后，自动在局域网广播配网信息
② 设备收到 UDP 广播后，自动提取 WiFi 配置
③ 设备保存并自动连接
```

> 💡 **推荐**: 如果您同时使用 PC 端监控软件，UDP 自动发现是最方便的方式。

---

### 3.4 连接成功

配网成功后，屏幕将显示：

```
┌───────────────────────┐
│  WiFi: 192.168.1.100  │   ← 设备的 IP 地址
│                       │
│  Server: 192.168.1.5  │   ← PC 端服务器地址
│                       │
│     Connecting...     │   ← 正在连接 PC 端
└───────────────────────┘
```

当 PC 端监控软件运行后，设备会自动连接并开始接收数据，屏幕切换为正常表情显示。

## 4. 日常使用指南

### 基本工作流程

```
┌──────────┐     WiFi      ┌──────────┐     TCP      ┌──────────┐
│  PC 端   │ ◄──────────► │  路由器  │ ◄──────────► │  宠物设备 │
│ 监控软件 │               │          │              │ (ESP32)  │
└──────────┘               └──────────┘              └──────────┘
     │                                                   │
     │ Python 桌面程序                                   │
     │ · 获取天气数据                                    │ · 显示表情
     │ · 统计 Token 用量                                 │ · 显示天气
     │ · 监控 CPU / 内存                                 │ · 显示资源
     │ · 推送到设备                                      │ · 触摸/旋钮交互
```

### 日常使用

1. **开机**: 用 USB 线接通电源，设备自动启动并连接 WiFi
2. **正常运行**: 设备自动从 PC 端接收数据，屏幕显示表情、天气和状态信息
3. **交互**: 通过触摸屏幕或旋转旋钮与宠物互动
4. **省电**: 无操作时设备自动进入省电模式，触摸或收到新数据时自动唤醒
5. **关机**: 拔掉 USB 线即可（无需特殊关机操作）

## 5. 屏幕显示说明

### 5.1 主界面布局

屏幕采用 **分区布局**，信息一目了然：

```
┌───────────────────────────┐
│                           │
│      [表情动画区域]       │  ← 32×32 像素，4 帧动画
│        32 × 32            │     每 200ms 切换一帧
│                           │
├───────────────────────────┤
│  🌡 26.5°C  💧 65%        │  ← 温度 / 湿度
│  🌤 晴     🌬 3.2m/s      │  ← 天气描述 / 风速
├───────────────────────────┤
│  🤖 Token: 12,345         │  ← AI Token 用量
│  💻 CPU: 23%  RAM: 1.2GB  │  ← 电脑资源占用
└───────────────────────────┘
```

### 5.2 四种表情状态

宠物会根据 AI 助手的工作状态自动切换表情：

| 表情 | 状态 | 含义 | 屏幕显示 |
|:----:|------|------|----------|
| 😊 | **空闲** (Happy) | AI 助手空闲等待中，一切正常 | 开心放松的像素小脸 |
| 😤 | **工作中** (Working) | AI 助手正在执行任务 | 专注认真的表情 |
| 😰 | **需授权** (Auth) | AI 助手需要您确认操作 | 紧张等待的表情 |
| 😴 | **离线** (Offline) | 未连接到 PC 端（超过 45 秒无数据） | 睡眠状态 + 显示时钟 |

> 💡 表情切换是自动的，您无需手动操作。

### 5.3 天气显示

天气数据每 **10 分钟**自动更新，显示内容包括：

| 项目 | 说明 | 示例 |
|------|------|------|
| 城市 | 当前定位城市 | 北京 |
| 温度 | 实时气温 | 26.5°C |
| 体感温度 | 人体感受温度 | 28.3°C |
| 湿度 | 相对湿度 | 65% |
| 天气描述 | 天气状况 | 晴 / 多云 / 小雨 |
| 天气图标 | 像素风格天气图标 | ☀️ / ⛅ / 🌧 |
| 风速 | 风速 | 3.2 m/s |

### 5.4 资源监控

| 项目 | 说明 | 数据来源 |
|------|------|----------|
| Token 用量 | AI 模型的 Token 消耗统计 | PC 端监控软件 |
| CPU 占用 | PC 的 CPU 使用率 | PC 端监控软件 |
| 内存占用 | PC 的内存使用量 | PC 端监控软件 |

> 数值变化时，屏幕上的数字会以 **弹簧动画** 平滑过渡，不会有突兀的跳变。

### 5.5 离线时钟

当设备与 PC 端断开连接超过 45 秒后，自动切换为 **离线模式**：

```
┌───────────────────────────┐
│                           │
│         😴                │  ← 睡眠表情
│                           │
│       14:32:05            │  ← 实时时钟（NTP 同步）
│                           │
└───────────────────────────┘
```

时钟基于 NTP 网络时间同步，精度高。即使断开 PC 端，只要 WiFi 连接正常，时间依然准确。

### 5.6 自定义像素动画

除了内置表情外，PC 端还可以推送 **自定义像素动画** 到设备上全屏播放：

```
┌───────────────────────────┐
│                           │
│   全屏像素动画播放中...   │  ← 32×32 像素，RGB565
│                           │
└───────────────────────────┘
```

播放自定义动画时：
- 内置表情暂停显示
- 按 **BOOT 按钮** 可停止播放，返回内置表情
- 详见 [第 12 章：自定义像素动画](#12-自定义像素动画)

## 6. 触摸与手势操作

设备屏幕支持 **电容触摸**，定义了 4 种手势：

| 手势 | 操作方式 | 声音反馈 | 振动反馈 | 功能 |
|:----:|----------|:--------:|:--------:|------|
| 👆 **单击** | 轻触屏幕一次 | 无 | 轻触感 | 唤醒屏幕 / 交互确认 |
| 👆👆 **双击** | 快速轻触两次 | 通知音 | 嗡嗡感 | 切换显示内容 / 特殊操作 |
| ✊ **长按** | 持续按住屏幕 >0.8 秒 | 警报音 | 强力振动 | 进入设置 / 更多选项 |
| 🤚 **接近** | 手掌靠近屏幕（不接触） | 无 | 无 | 唤醒屏幕 |

### 触摸使用技巧

- **触摸校准**: 设备启动时会自动校准触摸基线。请在启动过程中**不要触摸屏幕**，否则会影响触摸灵敏度
- **接近感应**: 手掌靠近屏幕（约 2-3cm）即可唤醒已休眠的屏幕，无需直接接触
- **消抖处理**: 设备内置 50ms 触摸消抖，快速连续触摸不会产生误操作
- **手势判定**:
  - 两次点击间隔 < 300ms → 判定为双击
  - 按住 > 800ms → 判定为长按
  - 单次轻触 → 判定为单击

## 7. 旋钮（编码器）操作

旋转编码器提供 **旋转** 和 **按压** 两种操作：

| 操作 | 方式 | 说明 |
|:----:|------|------|
| 🔄 **顺时针旋转** | 向右旋转旋钮 | 数值增加 / 向下滚动 / 调高亮度 |
| 🔄 **逆时针旋转** | 向左旋转旋钮 | 数值减少 / 向上滚动 / 调低亮度 |
| ⏺️ **按压** | 按下旋钮 | 确认选择 / 切换模式 |

> 💡 **编码器参数**: 每圈 24 个脉冲，带内部去抖滤波（步长=4），旋转手感精确流畅。

## 8. 物理按键说明

设备侧面有两个物理按键（需要用细针或卡针操作）：

| 按键 | 位置 | 功能 |
|:----:|------|------|
| **BOOT** | 底部小孔 | **短按**: 停止自定义像素动画，返回内置表情；屏幕休眠时短按唤醒屏幕（亮屏 15 秒） |
| **RST** | 底部小孔 | **短按**: 重启设备（等同于拔插电源） |

### BOOT 按键使用场景

```
场景 1: 屏幕休眠中
  按下 BOOT → 屏幕唤醒，显示 15 秒后自动休眠

场景 2: 正在播放自定义像素动画
  按下 BOOT → 停止动画，返回内置表情显示

场景 3: 正常显示内置表情
  按下 BOOT → 无操作（当前已是内置表情模式）
```

## 9. 声音与触觉反馈

### 9.1 声音效果

设备内置无源蜂鸣器，支持以下音效：

| 音效 | 触发时机 | 描述 |
|------|----------|------|

| **通知音** | 双击屏幕 | 短促的提示音 |
| **警报音** | 长按屏幕 | 连续短促的提醒音 |
| **OTA 进度音** | 固件升级中 | 进度提示音 |
| **OTA 成功音** | 升级完成 | 愉悦的成功音调 |

| **轻触** | 系统启动完成 | 轻柔的点击感 |

| **强击** | 长按屏幕 | 短促有力的振动 |

> 💡 如果未连接振动马达模块，触觉反馈功能自动跳过，不影响其他功能。

## 10. 智能省电系统

设备内置 **三级自动省电** 机制，兼顾续航和响应速度：

```
  正常运行                  30秒无数据                 60秒无数据              5分钟无数据
  (全速 50fps)             (亮度变暗 1fps)            (屏幕关闭 2fps)        (深度休眠)
       │                        │                         │                      │
  ─────┼────────────────────────┼─────────────────────────┼──────────────────────┼───►
       │                        │                         │                      │
  [全亮模式]              [自动变暗]                 [屏幕休眠]           [Light Sleep]
  CPU 240MHz              CPU 240MHz                 CPU 80MHz             CPU 80MHz
  WiFi 活跃               WiFi 活跃                  WiFi 省电             WiFi 省电
  帧率 50fps              帧率 1fps                  帧率 2fps             超低功耗
```

### 各阶段详解

| 阶段 | 触发条件 | 屏幕状态 | 功耗 | 唤醒方式 |
|:----:|----------|:--------:|:----:|----------|
| **正常** | 持续收到数据 | 全亮 | 正常 | — |
| **变暗** | 30 秒无数据 | 亮度降低 | 略低 | 收到新数据 / 触摸 / 接近 |
| **休眠** | 60 秒无数据 | 关闭背光 | 低 | 收到新数据 / 触摸 / BOOT 键 |
| **深度休眠** | 休眠后 5 分钟 | 关闭 | 极低 | 触摸唤醒 / WiFi 信标 |

> 📊 **功耗说明**: 深度休眠模式下，设备进入 ESP32 Light Sleep，功耗降至最低。触摸屏幕即可唤醒，响应时间 < 1 秒。

### 夜览模式（Night Shift）

设备根据 **NTP 网络时间** 自动调整屏幕色温：

| 时段 | 色温 | 说明 |
|------|------|------|
| 20:00 - 06:00 | 暖色调 | 自动减少蓝光，保护眼睛 |
| 06:00 - 20:00 | 正常色调 | 还原标准色彩 |

## 11. PC 端配套软件

### 11.1 概述

PC 端监控软件是一个 **Python 桌面程序**，运行在系统托盘中，负责：

4. **推送到设备** — 通过 TCP 将数据发送到宠物设备

## 12. 自定义像素动画

### 12.1 概述

除了内置的 4 种表情，您还可以通过 PC 端推送 **自定义像素动画** 到设备上全屏播放。

动画格式为 **PXL**（Pixel Animation），特点：
- 分辨率：**32×32** 像素
- 色彩：**RGB565**（65,536 色）
- 帧数：支持多帧动画
- 传输：支持分包传输，单包最大 16KB

### 12.2 PXL 文件格式

```
┌──────────┬──────────┬──────────────────────────────┐
│ Header   │ Frame    │ Pixel Data                   │
│ (12B)    │ Count(4B)│ (宽×高×2 × 帧数 字节)        │
└──────────┴──────────┴──────────────────────────────┘
```

| 字段 | 长度 | 说明 |
|------|------|------|
| Magic | 4B | 魔数标识 `PXL\0` |
| Version | 2B | 格式版本 |
| Width | 2B | 图像宽度（32） |
| Height | 2B | 图像高度（32） |
| Frame Count | 4B | 帧数量 |
| Pixel Data | 可变 | 每帧 32×32×2 = 2,048 字节 |

### 12.3 使用方法

1. **准备 PXL 文件**: 使用配套的转换工具将图片/动画转为 PXL 格式
2. **PC 端推送**: 通过 PC 端监控软件发送 `pixel_data` 消息（自动分包传输）
3. **开始播放**: 发送 `pixel_start` 指令，设备切换为全屏像素模式
4. **停止播放**: 
   - 按设备上的 **BOOT 按钮**
   - 或 PC 端发送 `pixel_stop` 指令

### 12.4 Python 转换工具示例

```python
import struct

def convert_to_pxl(image_data: bytes, width: int = 32, height: int = 32, frames: int = 1) -> bytes:
    """将 RGB565 像素数据转换为 PXL 格式"""
    header = b'PXL\x00'                    # Magic
    header += struct.pack('<H', 1)          # Version
    header += struct.pack('<HH', width, height)  # Size
    header += struct.pack('<I', frames)     # Frame count
    return header + image_data
```

## 13. 固件升级（OTA）

### 13.1 概述

设备支持 **无线固件升级**（Over-The-Air），无需拆机或连接 USB 线。升级通过 PC 端监控软件推送。

### 13.2 升级流程

```
PC端推送OTA ──► 设备收到 ──► 校验固件 ──► 写入Flash ──► 重启
                                                  │
                                              校验失败
                                                  │
                                           保留旧固件运行
```

### 13.3 升级过程中的设备表现

| 阶段 | 屏幕显示 | 声音 | 振动 |
|------|----------|------|------|
| 接收固件 | "OTA Updating..." + 进度条 | 进度提示音 | — |
| 写入 Flash | "Writing..." | — | — |
| 升级成功 | "OTA Success! Rebooting..." | 成功音 | — |
| 升级失败 | "OTA Failed!" | 失败音 | — |
| 重启 | 启动画面 | 启动音 | 启动振动 |

### 13.4 安全机制

- **双分区设计**: 设备使用 A/B 分区，升级失败自动回滚到旧版本
- **校验和验证**: 固件数据全程 MD5 校验，防止损坏
- **自动恢复**: 如升级后无法启动，设备自动恢复到上一个正常版本

> ⚠️ **升级注意事项**:
> - 升级过程中**不要断电**，否则可能导致固件损坏
> - 升级通常需要 1-3 分钟，取决于固件大小和网络速度
> - 升级完成后设备会自动重启

## 14. 常见问题与故障排除

### 设备相关

| 症状 | 可能原因 | 解决方法 |
|------|----------|----------|
| 开机后黑屏 | 未接电源 / USB 线损坏 | 更换 USB 线，确认电源指示灯亮 |
| 屏幕显示 "WiFi Failed" | WiFi 密码错误 / 信号弱 | 重新配网，靠近路由器 |
| 屏幕一直显示 "Connecting..." | PC 端软件未运行 | 启动 PC 端监控软件 `tray_app.py` |
| 表情一直是 😴（离线） | 与 PC 端断开 >45 秒 | 检查 WiFi 和 PC 端软件是否正常运行 |
| 触摸无反应 | 启动时触摸了屏幕（基线异常） | 重启设备，启动过程中不要触摸屏幕 |
| 旋钮旋转无反应 | 编码器线松动 | 检查编码器连接线是否插紧 |
| 没有声音 | 蜂鸣器未连接 / 被静音 | 检查蜂鸣器连接，在 PC 端检查静音设置 |
| 没有振动 | DRV2605L 未连接 | 检查 I2C 连接线；无振动马达不影响其他功能 |
| 屏幕频繁休眠 | 数据更新间隔过长 | 检查 PC 端是否正常运行，网络是否稳定 |

### PC 端相关

| 症状 | 可能原因 | 解决方法 |
|------|----------|----------|
| 启动报错 | 缺少依赖 | 运行 `pip install -r requirements.txt` |
| 天气获取失败 | Edge 浏览器未安装 / 版本过低 | 安装或更新 Microsoft Edge |
| 设备连不上 | IP 地址不匹配 | 确认设备和电脑在同一局域网 |
| Token 统计不准 | AI 框架未适配 | 检查 `config.json` 中的统计配置 |

### 配网相关

| 症状 | 可能原因 | 解决方法 |
|------|----------|----------|
| 找不到 "Pet-Setup" 热点 | 设备未进入配网模式 | 长按 BOOT 键重置，或等待超时自动重启 |
| 配网页面打不开 | IP 地址输入错误 | 确认输入 `http://192.168.4.1`（不是 https） |
| 配网超时 | 操作时间 >2 分钟 | 重启设备，2 分钟内完成配网操作 |
| 配网成功但连不上 | WiFi 信号弱 / 路由器限制 | 靠近路由器，检查路由器是否限制了设备接入 |
